# Resource-limit allowlist — check-resource-limits.ts (vector OOM PR #85 포스트모템 + CPU 단일축 편향 해소 가드)
#
# 규칙: 상주 워크로드(Deployment/DaemonSet/StatefulSet)의 main 컨테이너는 cpu·memory request + memory limit
#   필수(cpu limit은 비요구 — throttling 회피). CNPG CR도 스캔한다: Pooler는 pgbouncer 컨테이너,
#   Cluster는 spec.resources를 pseudo-container 'postgres'로 검사(키 형식 Cluster/<name>/postgres).
#   부득이 미설정해야 하면 아래에 한 줄로 등재한다(게이트 면제):
#       <Kind>/<name>/<container>   # 이유   (Cluster는 <container>=postgres)
#   '#' 이후는 주석. 형식 외 라인은 면제 항목이 아니다(문서로만 취급).
#
# ── 강제 면제(enforced exemptions) ──
#   현재 없음 — 로컬 소스 매니페스트의 전 상주 워크로드가 cpu·memory request + memory limit 보유(위반 0).
#
# ── 범위 밖(문서 전용, 게이트 미강제) — 의도적으로 limit 미설정인 알려진 컨테이너 ──
#   이들은 원격-helm 차트/벤더/k3s-시스템이라 소스 매니페스트에 없어 이 게이트가 스캔하지 못한다.
#   강제 대신 "왜 unlimited인가"를 여기 기록해 블라인드스팟을 가시화한다. 변경은 라이브로 확인.
#     - cert-manager controller/cainjector/webhook : 원격 helm 차트(platform/argocd/root/apps/cert-manager.yaml).
#         임계 컴포넌트라 too-tight limit이면 cert 발급/갱신 장애(self-induced OOM) 위험 → 노드 여유로 흡수.
#         (limit이 있는 워크로드는 ContainerMemoryNearLimit 알림이 OOM 전에 커버하나, unlimited는 미커버.)
#     - barman-cloud : 벤더 barman-plugin manifest(platform/cnpg/barman-plugin — 수정 금지).
#     - database/pg-1 plugin-barman-cloud : CNPG가 pg StatefulSet에 주입하는 사이드카(소스 매니페스트 부재, unlimited).
#     - edge/ts-traefik-* tailscale : tailscale operator가 만드는 proxy StatefulSet(런타임 생성·ProxyClass 미지정 → unlimited).
#         근본 조치 여지: ProxyClass CR 정의 후 defaultProxyClass 지정으로 proxy에 memory limit 부여.
#     - kube-system svclb-* (klipper-lb) : k3s 관리(우리 매니페스트 아님).
#
#   ── 스캔 제외(but limit 보유) — 아래는 unlimited가 아니라 charts/ 하위라 게이트가 못 보는 것뿐 ──
#     local-helm 상주(traefik·argocd·sealed-secrets·tailscale-operator)는 렌더가 charts/(스캔 제외)에 떨어지나
#     각 values(values-traefik.yaml 등)에 memory limit을 선언한다 — 회귀 시 게이트 미포착이라 values 직접 확인.
#     - cnpg-operator manager : 원격 helm 차트라 스캔 밖이나 limit 보유(platform/cnpg/operator/values.yaml
#         top-level resources 160Mi, 원장 cnpg-operator 행과 동기). 2026-07-06 이전엔 values가 umbrella용
#         네스팅으로 조용히 무시돼 실제 unlimited였다 — 회귀 시 라이브 deploy resources로 확인.
